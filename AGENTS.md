## Hardware Versions

The project supports two board revisions via the `HW_VERSION` build flag:

| Version | Buttons A/B | SD Card | Build command |
|---------|------------|---------|---------------|
| **v1** (original) | PA5 / PA4 | None | `make build-v1` / `pio run -e v1` |
| **v2** (current)   | PB0 / PB1 | SPI1 (PA4–PA7) | `make build` / `pio run -e v2` |

The pinout is managed in `lib/PHSI245_HAL/HAL.h` via `#if HW_VERSION == 1` / `#elif HW_VERSION == 2`.
Default is v2. To switch, change the `ENV` variable in the Makefile or pass `-e v1` / `-e v2` to `pio run`.

On v1 hardware, the bootloader displays an info screen indicating that storage is unavailable
(since there is no SD card slot). On v2, it proceeds to mount the SD card and launch the
file browser.

## Documentation

The OS library documentation lives in `os/doc/html/` and is generated by
[Doxygen](https://www.doxygen.nl/) from the source files in these directories:

- `lib/PHSI245_HAL/`
- `lib/phsi245_gfx/`
- `lib/phsi245_input/`
- `lib/phsi245_storage/`

**Regenerate the docs whenever any header or source file in those libraries
changes:**

```bash
doxygen os/Doxyfile
```

The generated output under `os/doc/html/` is committed to the repository and
must be kept in sync with the source.

## Bootloader

The main firmware (`src/main.cpp`) is the OS bootloader. It provides:
- A loading animation with progress bar on startup
- SD card mounting and FAT filesystem initialization
- A file browser (UP/DOWN to navigate, A to select, B to go back)
- File info view and simulated execution (displays the file header)

The bootloader uses all four OS libraries: HAL, gfx, input, and storage.
